Web Apps Mythbusting

3 years ago we heared that creating a 100% HTML5 Device was impossible

We proved them wrong!

Despite that, we keep hearing there are many things the Web can't do

Myth #1

I NEED TO CREATE A SPECIFIC VERSION OF THE APP FOR EVERY DEVICE/BROWSER

Responsive Design

ISN'T WRITE ONCE RUN ANYWHERE A REALITY TODAY?

Myth #2

THERE ARE NO APIs IN HTML5

Myth #3

HTML5 APPS DO NOT WORK OFFLINE

Plenty of options to make your app work offline

  • Application Cache: To store static resources for being used offline.
  • IndexedDB: Allows storing information for being used offline in a structured way.
  • Local Storage: To store small amounts of information in a non-structured way.

And more coming: Service Workers, intended to replace AppCache with a more flexible solution

Myth #4

WEB APPS CANNOT BE INSTALLED

In many platforms they are already first class citizens

  • Firefox for all the platforms
  • Chrome Desktop

And in many more platforms (iOS, Chrome Android...) supported via the fallback "Add to Homescreen"

You can do without an App Store

Finantial Times is the best example

... but if you really, really want to go trhough the AppStore you can always create a native version of your web app, e.g via PhoneGap

  • Use multiple device APIs

Is it that easy?

It's so easy it can be automated

What does the tool do?

  1. Inspect the app and create an appcache manifest
  2. Create the manifests for the different platforms
  3. Inject a library that allows user to install the app when browsing
  4. Host the app in a top level domain
